## Ownership contract

`TensorBuffer` shares an immutable `Arc<[u8]>` allocation with each exported
managed tensor; export does not clone bytes. A DLPack capsule owns exactly one
deleter responsibility until a consumer renames it to
`used_dltensor_versioned`. Import renames first, validates major version, rank,
shape, dtype, device, stride, offset, and reachable byte span, and calls the
producer deleter on every error path.

The safe crate surface denies `unsafe`; only `dlpack.rs` and
`dlpack_capsule.rs` opt into documented FFI operations. Non-host devices cannot
be converted to byte slices. Python rejects implicit `copy=True`, non-CPU
`dl_device`, and CPU stream requests. `Tensor.copy()` and
`tensor_copy_from_numpy` state allocation explicitly.

## Interoperability results

- Rust DLPack tests cover exact zero-copy pointers, scalar/strided metadata,
  negative strides, byte offsets, major mismatch, invalid dtype, negative shape,
  null data, and deleter behavior.
- Python binding suite: 61 tests passed with NumPy 2.4.3.
- NumPy consumed SpatialRust u8/u16/f32 tensors without copying and produced
  read-only arrays sharing the same pointer across exports.
- SpatialRust retained NumPy producer memory after the original Python reference
  was deleted.
- PyTorch 2.12.0+cpu consumed SpatialRust storage without copying and remained
  valid in the reverse producer-lifetime test.
