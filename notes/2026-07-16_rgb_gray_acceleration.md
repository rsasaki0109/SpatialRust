# Packed RGB8-to-gray acceleration — 2026-07-16

## Scope

This slice accelerates the existing safe `rgb_to_gray` and
`rgb_to_gray_into` APIs. It replaces the coarse Q8 coefficients with
OpenCV-compatible BT.601 Q14 coefficients, shares one output kernel between
allocation modes, enables CPU target-feature dispatch, and uses size-aware
Rayon scheduling: one-row tasks below 2,000 rows and eight-row blocks above.

Packed and strided input/output, metadata conversion to `ColorSpace::Gray`,
row padding, and Python `out=` identity remain supported. CPU execution does
not select a GPU or perform a device transfer.

## Native before / after

Criterion used seeded packed RGB8 inputs on the Windows host below. The old
reuse medians were 0.317 ms at VGA, 2.075 ms at 1080p, and 8.437 ms at 4K.
The accelerated medians were 0.054 ms, 0.195 ms, and 1.280 ms: about 5.9×,
10.6×, and 6.6× faster. Allocated medians improved from 0.756/5.162/17.838 ms
to 0.102/0.648/2.346 ms, about 7.4×–8.0× faster.

## OpenCV comparison

Focused paired/interleaved Python timings used OpenCV 4.13.0, CPython 3.12.10,
12 logical threads, OpenCL disabled, and seeded packed random RGB8 images.

| Profile | OpenCV allocate | SpatialRust allocate | OpenCV reuse | SpatialRust reuse | Outcome |
| --- | ---: | ---: | ---: | ---: | --- |
| VGA | 0.062 ms | 0.107 ms | 0.061 ms | 0.074 ms | OpenCV leads |
| 1080p | 0.850 ms | 0.825 ms | 0.215 ms | 0.232 ms | SpatialRust allocate 1.03× faster |
| 4K | 2.452 ms | 2.338 ms | 1.250 ms | 1.287 ms | SpatialRust allocate 1.05× faster |
| 8K | 7.400 ms | 7.630 ms | 5.885 ms | 5.754 ms | SpatialRust reuse 1.02× faster |

Canonical exact-pixel fractions were 99.72%–99.74%. Three hundred randomized
cases, including non-contiguous NumPy inputs, retained maximum absolute error
1. The focused receipt is generated by
`bench/opencv_rgb_gray_comparison/performance.py`; the authoritative local
JSON for this run was `target/opencv-rgb-gray-performance-final.json`.

## Boundary

The OpenCV wins are limited to allocated 1080p/4K and caller-owned 8K on the
named host and timing scope. OpenCV remains faster at VGA and for caller-owned
1080p/4K. Epic 115 continues with packed nearest/area resize and fused
resize-to-gray/CHW paths.
