# High-precision 7×7 Gaussian path — 2026-07-16

## Scope

This slice completes Epic 116C. Calls to `gaussian_blur_u8` and
`gaussian_blur_u8_into` with a 7×7 axis now use normalized Q15 kernels, a
workspace-owned `u32` horizontal image, and `u64` vertical accumulation with one
final rounded conversion. The caller-owned path writes directly to the supplied
output and no longer allocates a generic f64 image and copies it back.

The existing Q8 3×3/5×5 engine is unchanged. Kernel cache keys include the
fixed-point precision, so mixed 7×3 and 7×5 calls cannot reuse coefficients from
the Q8 path. Input ownership, explicit workspace reuse, CPU-only execution, and
`#![deny(unsafe_code)]` remain intact.

## Accuracy

Three hundred Rust property cases used RGB8 dimensions from 1 through 32,
sigma values from 0.3 through 4.0, and packed or strided input. The Q15 output
stayed within one `u8` level of the generic f64 implementation and retained
workspace capacity across a repeated call.

The Python extension was rebuilt in release mode. Three hundred randomized
7×7 RGB8 cases, including reversed non-contiguous NumPy input, were compared
with OpenCV 4.13 `GaussianBlur` using sigma 2.0 and Reflect101. The measured
maximum absolute error was 2/255, meeting the documented gate.

## Native measurement

Criterion measured packed RGB8, sigma 2.0, Reflect101, caller-owned output on
Windows 10, Intel Core i7-9750H, 12 logical processors, and the default Rayon
thread policy. These short local runs used 500 ms warm-up, 1 s requested
measurement time, and 10 samples.

| Profile | Generic f64 allocate | Q15 workspace reuse | Improvement |
| --- | ---: | ---: | ---: |
| VGA | 21.520 ms | 11.819 ms | 1.82× |
| 1080p | 160.94 ms | 17.256 ms | 9.33× |
| 4K | 688.47 ms | 65.119 ms | 10.6× |

This comparison is only against SpatialRust's previous generic allocation path
for the named workload. It does not claim a standalone OpenCV performance win.
